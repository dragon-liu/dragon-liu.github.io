c++中的代码重用

1.包含对象成员的类

  • 通常用于建立has-a关系的c++技术是组合(包含)，即创建一个包含其他类对象的类

  • 将typedef放在private当中的话类方法和友元函数可以使用，即在该类的实现中可以使用它，但在类外面不能使用

  • c++和约束：c++包含让程序员能够限制程序结构的特性——使用explicit防止单参数构造函数的隐式转换，使用const防止修改数据。根本原因是在编译阶段出现错误优于在运行阶段出现错误

  • 初始化顺序：当初始化列表包含多个项目时，这些项目被初始化的顺序为它们被声明的顺序，而不是它们在初始化列表中被声明的顺序

2.私有继承——-P543

  • 使用私有继承，基类的公有成员和保护成员都将成为派生类的私有成员。这意味着基类方法将不会成为派生对象公有接口的一部分，但可以在派生类的成员函数中使用它们。即与包含有相同特性：获得实现，但不继承接口

  • 初始化基类组件：对于继承类，新版本的构造函数将使用成员初始化列表语法，它使用类名而不是成员名来标识构造函数

  • 访问基类的方法：私有继承使得能够使用类名和作用域解析符来调用基类的方法

  • 访问基类的友元函数：用类名显示地限定函数名不适合于友元函数，这是因为友元不属于类。但可以通过显示地转换为基类来调用正确的函数。详见P546

  • 使用包含还是私有继承：由于两者均可用于建立has-a关系，如何选用。大多数c++程序员倾向于使用包含；如果新类需要访问原有类的保护成员，或需要重新定义虚函数，则应使用私有继承

  • 各种继承方式详见P550的表

  • 使用using重新定义访问权限：假设要让基类的方法在派生类外面可用——P550

    • 方法一：定义一个使用该基类方法的派生类方法
    • 方法二：将函数调用包装在另一个函数调用中，即使用一个using声明来指出派生类可以使用特定的基类成员，即使采用的是私有派生。注意，using声明只是用成员名，没有圆括号，函数特征表和返回类型。而且using声明只适用于继承，而不适用于包含

3.多重继承

  • MI描述的是有多个直接基类的类。与单继承一样，公有MI表示的也是is-a关系，且要注意必须使用public限定每一个基类

  • MI可能带来的两个主要问题：从两个不同的基类继承同名方法；从两个或更多相关基类那里继承同一个类的多个实例

  • c++引入多重继承的同时，引入了一种新技术，虚基类

  • 有关MI的其他问题：

    • 混合使用虚基类和非虚基类：当类通过多条虚途径和非虚途径继承某个特定的基类时，该类将包含一个表示所有的虚途径的基类子对象和分别表示各条非虚途径的多个基类子对象
    • 虚基类和支配：如果使用的是虚基类，若某个名称优先于其他所有名称，则使用它时，即便不适用限定符，也不会导致二义性。示例见P566

  • MI总结见P567

4.类模板

1.定义类模板

  • c++的类模板为生成通用的类声明提供了一种更好的办法。即能够将类型名作为参数传递给接收方来建立类或函数。和模板函数一样，模板类以下面这样的代码开头：template .尖括号中的内容相当于函数的参数列表。可以把关键字class看做是变量的类型名，该变量接受类型作为其值，但这里使用class并不意味着Type必须是一个类，而只是表明Type是一个通用的类型说明符。较新的实现可以用typename替换class

  • 注意：不能将模板成员函数放在独立的实现文件中。由于模板不是函数，它们不能单独编译，模板必须与特定的模板实例化请求一起使用。为此，最简单的方法是将所有模板信息放在一个头文件中，并在要使用这些模板的文件中包含该头文件

3.深入探讨模板类
  • 可以创建形如Stack st;的指针栈，但若不对程序做大修改则无法满足要求
  • 正确使用指针栈的方法之一：让调用程序提供一个指针数组，其中每个指针都指向不同的字符串。把这些指针放在栈中是有意义的，因为每个指针指向不同的字符串。注意,创建不同指针是调用程序的职责，而不是栈的职责——详见P573
4.数组模板示例和非类型参数———详见P577

  • 首先介绍一个允许指定数组大小的简单数组模板。一种方法是在类中使用动态数组和构造函数参数来提供元素数目；另一种方法是使用模板参数来提供常规数组的大小。

  • 注意这种模板头：template

  • 模板多功能性：可以将用于常规类的技术用于模板类。模板类可以用做基类，也可以用作组件类，还可以用作其他模板的类型参数

  • 使用多个类型参数：模板可以包含多个类型参数，详见P580

  • 模板的具体化：

    • 隐式实例化：即声明一个或多个对象，指出所需的类型，而编译器使用通用模板提供的处方生成具体地类定义
    • 显式实例化：当使用template并指出所需类型类声明类时，编译器将生成类声明的显式实例化。声明必须位于模板定义所在的名称空间中
    • 显式具体化：是特定类型的定义。有时候，可能需要在为特殊类型实例化时，对模板进行修改，使其行为不同
    • 部分具体化：c++还允许部分具体化，即部分限制模板的通用性。例如，部分具体化可以给类型参数之一指定具体的类型——详见P583

  • 成员模板：模板可用作结构，类或模板类的成员。详见P584

  • 还有更深入的有关友元的探讨详见P588
